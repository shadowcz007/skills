# 套餐设计助手

帮助你基于人群和应用场景设计产品套餐/定价方案。

## 触发条件

当用户想要：
- 设计套餐方案
- 制定定价策略
- 分析目标用户群体
- 定义产品使用场景
- 基于人群和场景拆分套餐

**不要**在以下情况触发：
- 已经明确知道具体定价参数的情况
- 需要直接计算价格的情况

---

## 套餐设计框架

### 一、人群划分维度

从以下4个维度分析目标用户：

| 维度 | 分类选项 |
|------|----------|
| **使用频率** | 轻度（<50次/天）、中度（50-200次/天）、高频（>200次/天） |
| **使用目的** | 学习、个人办公、专业工作、商业变现 |
| **专业程度** | 小白/学生、初级从业者、中高级专业人士 |
| **付费能力** | 个人免费版、付费个人版、企业版 |

### 二、应用场景维度

| 维度 | 分类选项 |
|------|----------|
| **任务复杂度** | 简单问答 → 短代码生成 → 完整项目开发 |
| **单次消耗量** | 轻量（<1k tokens）→ 中等（1-5k tokens）→ 重度（>5k tokens） |
| **实时性要求** | 异步处理 → 同步响应 |
| **并发需求** | 单用户 → 小团队 → 企业级 |

### 三、套餐要素

每个套餐需要定义：

1. **请求次数** - 最显性的限额
2. **tokens配额** - 隐性消耗（输入/输出分开）
3. **并发数** - 多人共用能力
4. **价格** - 基于成本+价值定价

## 设计步骤

1. **先定人群** → 明确目标用户是谁
2. **再定频率** → 估算用户会用多勤
3. **算出消耗量** → 频率 × 单次消耗 = 总消耗
4. **匹配定价** → 根据消耗量设计套餐额度
5. **给场景示例** → 让用户自己对号入座

---

## 定价策略参考

```
价格 = 基础成本 + 价值溢价

基础成本 = 请求次数成本 + tokens成本 + 并发成本
价值溢价 = 目标人群付费能力 × 使用场景紧迫度
```

---

## 注意事项

- 请求次数是最容易理解的限额，要放在第一位说明
- tokens是隐性消耗，要提醒用户注意
- 并发决定是否能多人共用/团队使用
- 给出具体的场景示例，让用户有参照
- 相邻套餐之间要有明显的差异化
